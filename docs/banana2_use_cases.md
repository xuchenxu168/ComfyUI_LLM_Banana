# Banana2 实战案例集（分模板场景）

本文件提供可直接套用的实战场景，覆盖 Banana2-提示词模板节点的主要模板。每个案例包含：适用模板、输入要点、参数建议、步骤与示例提示词，便于快速上手并稳妥产出。

## 使用建议（通用）
- 明确目标：写实 vs 风格化；生成 vs 编辑；单图 vs 多图。
- 输入尽量结构化：主体、环境、光照、构图、风格、版式、镜头等。
- 多图时优先 1–4 张参考图；强调一致的风格与色板。
- 参数：`temperature` 决定创意幅度（越高越大胆），`max_output_tokens` 决定输出细致程度。

## 案例 1：电商产品商业摄影
- 模板：`生成-产品模型和商业摄影`
- 目标：生成电商/广告级产品图，突出材质与构图稳定性。
- 输入要点：产品类型、材质/表面、背景/场景、灯光布置、镜头与角度。
- 参数建议：`temperature 0.3–0.6`，`max_output_tokens 512–768`
- 步骤：
  - 选择模板 → 填写产品细节 → 指定光线与构图 → 设置镜头参数。
- 示例提示词：
  - “不锈钢保温杯，纯白背景，三点布光，俯拍 45°，边缘高光清晰，16:9。”

## 案例 2：风格化插画与贴纸
- 模板：`生成-风格化插画和贴纸`
- 目标：卡通/极简/动漫等风格的贴纸或插画，支持透明背景。
- 输入要点：风格、主体、关键特征、色彩、线条与阴影风格、背景透明。
- 参数建议：`temperature 0.6–0.9`，`max_output_tokens 512`
- 示例提示词：
  - “极简扁平风格的柠檬贴纸，粗线条，明亮黄白配色，透明背景。”

## 案例 3：创意海报（中英文文字）
- 模板：`生成-图片中的文字` 或 `创意-海报设计`
- 目标：在画面中呈现明确的文字内容（中英文均可）。
- 输入要点：主视觉、主副标题（引号标注）、字体、版式、色板、层级。
- 参数建议：`temperature 0.4–0.7`，`max_output_tokens 768–1024`
- 示例提示词：
  - “A creative poster for spring sale，主标题“SPRING SALE”，副标“限时折扣”，现代无衬线字体，居中版式，红白配色，大量留白。”

## 案例 4：多图高级合成（服饰替换）
- 模板：`编辑-高级合成(多图组合)`
- 目标：将 A 图的服饰自然合成到 B 图的模特身上。
- 输入要点：两张参考图（服饰图、模特图）、褶皱/阴影匹配、透视与光线一致性。
- 参数建议：`temperature 0.4–0.7`，`max_output_tokens 768–1024`
- 示例提示词：
  - “将第一张图的连衣裙合成到第二张图的模特身上，匹配褶皱与阴影，保证衣料纹理真实，自然融合。”

## 案例 5：高保真细节保留（添加 LOGO）
- 模板：`编辑-高保真细节保留`
- 目标：在不损伤纹理的情况下添加徽标/文字。
- 输入要点：关键区域、保留细节项、编辑目标、质量稳定性要求。
- 参数建议：`temperature 0.2–0.5`，`max_output_tokens 512–768`
- 示例提示词：
  - “在黑色 T 恤胸前添加白色品牌 Logo，保持布料纹理与光照一致，边缘过渡自然。”

## 案例 6：风格迁移（赛博朋克）
- 模板：`编辑-风格迁移`
- 目标：将参考图风格迁移到目标图，保留结构与关键元素。
- 输入要点：源风格图、目标内容图、迁移强度、保留结构细节。
- 参数建议：`temperature 0.5–0.8`，`max_output_tokens 512–768`
- 示例提示词：
  - “将室内照片迁移为赛博朋克霓虹风格，保留空间结构与材质，增强蓝紫霓虹与雾气。”

## 案例 7：角色一致性（多视角）
- 模板：`编辑-角色一致性(360度)`
- 目标：同一角色的不同视角/姿势，身份与风格保持一致。
- 输入要点：角色特征、参考图、目标视角、风格一致、表情控制。
- 参数建议：`temperature 0.4–0.7`，`max_output_tokens 768–1024`
- 示例提示词：
  - “根据正面参考图生成该角色的侧面视图，保持服饰与五官一致，微笑表情，暖色室内光照。”

## 案例 8：连续艺术（三分格短漫画）
- 模板：`生成-连续艺术(漫画分格)`
- 目标：三分格起承转叙事，风格连贯自然过渡。
- 输入要点：格数/结构、场景推进、角色动作与表情、风格一致性。
- 参数建议：`temperature 0.5–0.8`，`max_output_tokens 768–1024`
- 示例提示词：
  - “三分格短故事：起（角色遇见神秘包裹）-承（打开发现奇异装置）-转（装置启动，光线爆发），保持角色一致与色调连贯。”

## 故障排除与优化
- 提示词太简单 → 提高 `temperature` 或补充细节；适度增加 `max_output_tokens`。
- 风格不统一 → 明确风格参考图与色板；在提示中强调“一致性”。
- 文字错乱 → 中文/英文分别用引号标注；指定字体与版式层级。
- 合成不自然 → 强调阴影/透视/反射匹配；必要时降低 `temperature`。
- 结果太短 → 增大 `max_output_tokens`；按结构化要点重写输入。
